В Java **примитивные типы** (primitive types) — это базовые встроенные типы данных, которые не являются объектами и хранятся непосредственно в памяти (не через ссылку). Они предопределены языком и обозначаются зарезервированными ключевыми словами.

Всего в Java **8 примитивных типов**, разделённых на 4 группы:


### 1. Целые числа
Хранят целочисленные значения со знаком.

- **`byte`**
    - Размер: 1 байт (8 бит).
    - Диапазон: от -128 до 127.
    - Используется для экономии памяти в больших массивах (например, при работе с потоками данных).

- **`short`**
    - Размер: 2 байта (16 бит).
    - Диапазон: от -32\,768 до 32\,767.
    - Применяется редко, в основном для экономии памяти.

- **`int`**
    - Размер: 4 байта (32 бита).
    - Диапазон: от  -2^{31}  до  2^{31}-1  (примерно от  -2,147,483,648  до  2,147,483,647 ).
    - Самый распространённый тип для целых чисел. Виртуальная машина Java оптимизирована под `int`.

- **`long`**
    - Размер: 8 байт (64 бита).
    - Диапазон: от  -2^{63}  до  2^{63}-1 .
    - Используется для очень больших чисел (например, временных меток).
    - Литералы требуют суффикса `L`: `long timestamp = 1672531200000L;`.

### 2. Числа с плавающей точкой
Хранят дробные значения.

- **`float`**
    - Размер: 4 байта (32 бита).
    - Точность: ~7–8 значащих цифр.
    - Используется там, где важна экономия памяти (например, графика).
    - Литералы требуют суффикса `f` или `F`: `float rate = 3.14f;`.

- **`double`**
    - Размер: 8 байт (64 бита).
    - Точность: ~15–17 значащих цифр.
    - Стандартный выбор для математических вычислений (включая финансовые, если не требуется абсолютная точность).


### 3. Логический тип
- **`boolean`**
    - Размер: 1 бит (но в памяти может занимать больше из‑за выравнивания).
    - Значения: `true` (истина) или `false` (ложь).
    - Не эквивалентен числам `0`/`1` (в отличие от некоторых других языков).


### 4. Символьный тип
- **`char`**
    - Размер: 2 байта (16 бит).
    - Хранит один символ Unicode (в одинарных кавычках).
    - Может быть задан:
        - по символу: `char grade = 'A';`
        - по коду Unicode: `char symbol = '\u0436';` (кириллическая «ж»).


### Важные особенности
1. **Значения по умолчанию** (для полей класса):
    - `byte`, `short`, `int`, `long` → `0`;
    - `float` → `0.0f`;
    - `double` → `0.0d`;
    - `char` → `'\u0000'`;
    - `boolean` → `false`.

2. **Литералы** (константы в коде):
    - Целые числа по умолчанию имеют тип `int`. Для `long` нужен суффикс `L`.
    - Дробные числа по умолчанию имеют тип `double`. Для `float` нужен суффикс `f`.

3. **Классы‑обёртки**  
   Каждому примитивному типу соответствует класс‑обёртка (для работы с объектами):
    - `byte` → `Byte`;
    - `short` → `Short`;
    - `int` → `Integer`;
    - `long` → `Long`;
    - `float` → `Float`;
    - `double` → `Double`;
    - `boolean` → `Boolean`;
    - `char` → `Character`.

4. **Автоупаковка и распаковка** (autoboxing/unboxing):
    - Компилятор автоматически преобразует примитивы в обёртки и обратно:
      ```java
      Integer boxed = 42;        // автоупаковка
      int unboxed = boxed;      // распаковка
      ```

### Почему не использовать `double` для денег?
Тип `double` хранит числа в формате IEEE 754, что может приводить к ошибкам округления. Например:
```java
System.out.println(0.1 + 0.2); // Выведет: 0.30000000000000004
```
Для финансовых расчётов используйте `BigDecimal`.

В Java **операторы сравнения** (relational operators) служат для сопоставления двух значений. Все они являются **бинарными** (требуют два операнда) и возвращают результат типа `boolean`: `true` (если условие выполняется) или `false` (если не выполняется).


### Список операторов сравнения

1. **`==`** — равенство  
   Проверяет, равны ли два значения.
   ```java
   int a = 5, b = 5;
   boolean result = (a == b); // true
   ```

2. **`!=`** — неравенство  
   Проверяет, различаются ли два значения.
   ```java
   boolean result = (a != b); // false
   ```

3. **`>`** — больше  
   Проверяет, больше ли левый операнд, чем правый.
   ```java
   boolean result = (10 > 5); // true
   ```

4. **`<`** — меньше  
   Проверяет, меньше ли левый операнд, чем правый.
   ```java
   boolean result = (3 < 7); // true
   ```

5. **`>=`** — больше или равно  
   Проверяет, больше ли или равен левый операнд правому.
   ```java
   boolean result = (5 >= 5); // true
   ```

6. **`<=`** — меньше или равно  
   Проверяет, меньше ли или равен левый операнд правому.
   ```java
   boolean result = (4 <= 6); // true
   ```

### Важные нюансы и правила использования

1. **Типы операндов**
    - Операторы `>`, `<`, `>=`, `<=` применяются **только к числовым типам** (`byte`, `short`, `int`, `long`, `float`, `double`) и `char`.
    - Операторы `==` и `!=` работают **со всеми типами**, включая ссылочные (объекты).

2. **Сравнение объектов vs примитивов**
    - Для примитивов `==`/`!=` сравнивают **значения**.
    - Для объектов `==`/`!=` сравнивают **ссылки** (т. е. указывают ли две переменные на один и тот же объект в памяти).
      ```java
      String s1 = new String("hello");
      String s2 = new String("hello");
      System.out.println(s1 == s2); // false (разные объекты)
      System.out.println(s1.equals(s2)); // true (одинаковое содержимое)
      ```
   → Для сравнения содержимого объектов используйте метод `equals()`.

3. **Сравнение с `null`**
    - `==` и `!=` допустимы для проверки на `null`:
      ```java
      if (obj != null) {
          // безопасно использовать obj
      }
      ```
    - Попытка сравнить `null` с помощью других операторов (`>`, `<` и т. п.) вызовет `NullPointerException`.

4. **Приоритет операций**  
   Операторы сравнения имеют **более низкий приоритет**, чем арифметические операции, но **выше**, чем логические (`&&`, `||`).
   ```java
   boolean result = (5 + 3 > 2 * 4); // (8 > 8) → false
   ```

5. **Запрет неявного приведения типов**  
   Нельзя сравнивать несовместимые типы (например, `String` и `int`):
   ```java
   boolean result = ("5" == 5); // Ошибка компиляции!
   ```

6. **Особенности для `float`/`double`**  
   Из‑за погрешностей представления чисел с плавающей точкой сравнение через `==` может давать неочевидные результаты:
   ```java
   double a = 0.1 + 0.2;
   double b = 0.3;
   System.out.println(a == b); // false!
   ```
   → Для таких случаев лучше использовать проверку с эпсилоном:
   ```java
   boolean equals = Math.abs(a - b) < 1e-10;
   ```

### Примеры использования в условных конструкциях

Операторы сравнения чаще всего применяются в `if`, `while`, `for`:

```java
int score = 85;

if (score >= 90) {
    System.out.println("Отлично");
} else if (score >= 75) {
    System.out.println("Хорошо");
} else {
    System.out.println("Нужно подтянуться");
}
```

```java
while (counter < 10) {
    System.out.println("Итерация: " + counter);
    counter++;
}
```

**Итог:**
- Операторы сравнения — основа условной логики в Java.
- Всегда учитывайте тип данных и контекст (примитивы vs объекты).
- Для объектов используйте `equals()`, а не `==`.
- Будьте осторожны при сравнении чисел с плавающей точкой.

**Тернарный условный оператор** в Java — это оператор, который позволяет записать условное выражение в компактной форме. Он состоит из трёх частей: условия, значения при истине и значения при лжи. Синтаксис оператора:

```java
условие ? значение_если_истина : значение_если_ложь
```

**Пример использования:**

```java
int a = 10;
int b = 20;
String result = (a > b) ? "a больше b" : "b больше или равно a";
System.out.println(result); // Выведет: "b больше или равно a"
```

**Объяснение:**
1. Условие `(a > b)` проверяется.
2. Если условие истинно, возвращается значение после `?` (`"a больше b"`).
3. Если условие ложно, возвращается значение после `:` (`"b больше или равно a"`).

**Применение:**
- Используется для упрощения кода, когда нужно выбрать одно из двух значений в зависимости от условия.
- Часто применяется в присваиваниях, например:
  ```java
  int max = (a > b) ? a : b;
  ```

Альтернативное решение задачи:
```java
import java.util.Scanner;
public class BonusSystem {
    public static void main(String[] args) {
    }

    Scaner scaner = new Scaner(System.in);

    //Ввод данных
        System.out.print("Введите сумму покупки");
    double purchaseAmount = scaner.nextDouble();
        System.out.print("Является ли пользователь зарегистрированным? (Да/Нет): ");
    String isRegistered = scaner.next().toLowerCase();

    // Расчёт бонусного бала
    double bonusPoints = calculateBonusPoints(purchaseAmount, isRegistered.equals("Да"));

    //Вывод результата
        System.out.println("Начисленно бонусных баллов: "+bonusPoints);
}


private static double calculateBonusPoints ( double puchaseAmount, boolean isRegistered){
    double bonusRate = isRegistered ? 0.03 : 0.01; // 3% для зарегистрированных, 1% для незарегистрированных
    double bonusPoints = puchaseAmount * bonusRate;

    // ограничение максимального бонуса
    return Math.min(bonusPoints, 500);
} 
```
### Ещё вариант
```java
public static int calculateBonus(int amount, boolean registeredUser) {
final int MAX_BONUS = 500;
int percent = registeredUser ? 3 : 1;

}

public static void main(String[] args) {
```
